## What this is

TileCam is a camera-management & live-streaming app for Apple platforms. It connects
to go2rtc / WebRTC streams and shows live camera feeds on **iPhone/iPad** and a
companion **Apple Watch** app, with motion analytics, audio metering, and
picture-in-picture. Repo is named `GlassView`; the product/app is `TileCam`.

- **Platforms:** iOS 17+ (iPhone, iPad, Mac Catalyst), watchOS 10+. **Dark mode only.**
- **Stack:** SwiftUI only (no UIKit), WebRTC (stasel/WebRTC 140.0.0) via SPM.
- **Build system:** xcodegen — `project.yml` is the source of truth, `TileCam.xcodeproj`
  is generated. Bundle IDs: `works.rainn.tilecam`, `works.rainn.tilecam.watchkitapp`.

## Architecture map

- `GlassView/App/` — app entry (`TileCamApp.swift`)
- `GlassView/Models/` — data models (`Stream.swift`)
- `GlassView/Services/` — session, WebRTC, go2rtc, audio, and motion analytics
  (`PhoneSessionManager`, `WebRTCClient`, `Go2RTCService`, `PiPManager`,
  `EulerianMagnifier`, `MotionAnalyzer`, `MotionFlowAnalyzer`, `IntensityHeatmapAnalyzer`)
- `GlassView/Views/` + `Views/Components/` — SwiftUI UI (`ContentView`, `StreamTileView`, …)
- `TileCamWatch/` — the watchOS app (`WatchSessionManager` handles iPhone↔Watch sync)

## Build & verify (READ THIS BEFORE BUILDING)

**Preference: skip builds unless you are deliberately verifying.** Builds are slow.
When you DO build, **target the physical iPad — NEVER the simulator.**

- Regenerate the Xcode project after adding/removing/renaming source files:
  `xcodegen generate`
- Verify on device (the real check) — current paired iPad is "iPad (3)". Device builds
  need `-allowProvisioningUpdates` AND an Apple ID configured in Xcode → Settings →
  Accounts (command-line signing can't generate profiles without it):
  `xcodebuild -project TileCam.xcodeproj -scheme TileCam -destination 'platform=iOS,id=00008027-000418CE2E0B802E' -allowProvisioningUpdates build`
  (Confirm the live UDID with `xcrun devicectl list devices` — it changes per device.)
- Signing-free compile check (no device needed):
  `xcodebuild -project TileCam.xcodeproj -scheme TileCam -sdk iphoneos -configuration Debug build CODE_SIGNING_ALLOWED=NO`
- Watch target: `xcodebuild -target TileCamWatch -sdk watchos build`

> A PostToolUse hook auto-runs `xcodegen generate` when a `.swift` file is written, so
> new files enter the build graph automatically. The hook does NOT run a full build —
> that is the **verifier** agent's deliberate job (see below).

## Conventions

- **No tests / no CI yet.** The verification loop is currently human + the `verifier`
  agent building on device. Treat "it compiles on the iPad" as the bar for "done."
- Match the surrounding SwiftUI idiom. No UIKit unless unavoidable.
- New source files go in the right folder (xcodegen picks them up by directory).
- Don't hand-edit `TileCam.xcodeproj/project.pbxproj` — it's regenerated by xcodegen.
- Commit messages: imperative, one feature/fix per commit (matches existing history).

## Design system

All UI work must conform to **`.impeccable.md`** (dark glassomorphism, opacity-as-hierarchy,
spring-loaded interactions, 44pt touch targets, the documented spacing/radius/duration
tokens). The `design-critic` agent and the `critique`/`audit`/`polish` skills enforce it.

## How we work — the agent team

This project is run by a human **orchestrator** directing a team of specialist agents.
See `.claude/ORCHESTRATION.md` for the full playbook. Two loops:

- **`/feature`** — build something: spec → plan → (gate) → implement → review → verify →
  design → (gate) → commit.
- **`/bug`** — fix something: reproduce → diagnose root cause (parallel, read-only) →
  (gate) → minimal fix → verify it's actually gone.

The roles live in `.claude/agents/`:

- `product` — turns intent into a spec with acceptance criteria (read-only)
- `architect` — implementation plan + approach + files to touch (read-only)
- `diagnostician` — finds a bug's root cause; reproduce/hypothesize/confirm (read-only)
- `implementer` — the only role that writes code
- `reviewer` — adversarial review in fresh context (read-only)
- `verifier` — builds on the iPad, reports evidence not assertions
- `design-critic` — conformance to `.impeccable.md` (read-only)
